TokenViz Lite 开发思路详解

不是"按顺序写代码"，而是"每一层为下一层准备什么"

--------------------------------------------------------------------------------
总体思路：自底向上，逐层验证
Layer 1  数据层    → 定义"存什么"
Layer 2  代理框架  → 定义"怎么拦截"
Layer 3  核心功能  → 定义"怎么提取和记录"
Layer 4  厂商适配  → 定义"各家有什么不同"
Layer 5  查询 API  → 定义"怎么查出来"
Layer 6  前端仪表板 → 定义"怎么展示"

每一层做完都能独立验证，不是写到第 6 层才第一次看到效果。
--------------------------------------------------------------------------------
Layer 1：数据库 — 先定义"终点长什么样"
为什么最先做数据库
数据库表就是你的"合约"——它定义了所有上游代码必须产出的数据格式，和所有下游代码必须消费的数据格式。先定表，就是先定合约。后面的代理、提取、查询全部围着这个合约转。
具体步骤

Layer 3：核心功能 — 非流式提取 + 流式尾捕获
为什么第三做核心功能
代理框架跑通后，你有了一条"管道"——数据从客户端流向上游再流回来。现在你要在管道上装一个"传感器"，记录流过的数据。
开发顺序：先非流式，再流式
非流式是最简单的情况——整个响应是一个完整 JSON，直接读 usage 字段。先把这个跑通，确认数据库写入没问题，再处理流式的复杂情况。

import pool from '../db/connection';

export async function calculateCost(
  provider: string,
  model: string,
  usage: CapturedUsage
): Promise<{ input_cost: number; output_cost: number; total_cost: number }> {
  const result = await pool.query(
    'SELECT * FROM model_pricing WHERE provider = $1 AND model = $2 AND is_active = true',
    [provider, model]
  );
  
  if (result.rows.length === 0) {
    // 未知模型：记录但不算错
    return { input_cost: 0, output_cost: 0, total_cost: 0 };
  }
  
  const pricing = result.rows[0];
  const MTok = 1_000_000;
  
  const input_cost = (usage.prompt_tokens * pricing.input_price / MTok)
                   + (usage.cache_creation_tokens * pricing.cache_write_price / MTok);
  const output_cost = usage.completion_tokens * pricing.output_price / MTok;
  const cache_cost = usage.cache_read_tokens * pricing.cache_read_price / MTok;
  
  return {
    input_cost: input_cost + cache_cost,
    output_cost,
    total_cost: input_cost + cache_cost + output_cost,
  };
}

3.2 流式尾捕获
第 1 步：写 streaming.ts — 核心模块
这是整个项目最关键的文件。开发思路是：
1. 转发优先 — 每个 chunk 必须立刻转发，不能等解析完再转
2. 顺便提取 — try-catch 包裹，解析失败不影响转发
3. 流结束写库 — 所有 chunk 处理完后，合并 usage 写库


import { ServerResponse } from 'http';
import { CapturedUsage } from '../types';

export async function handleStreaming(
  upstream: Response,
  clientRes: ServerResponse,
  provider: string,
  model: string,
  startTime: number,
  onRequestEnd: (usage: CapturedUsage | null, ttft: number | null) => void
) {
  // 1. 设置 SSE 响应头
  clientRes.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    'Connection': 'keep-alive',
  });

  let capturedUsage: CapturedUsage | null = null;
  let firstTokenTime: number | null = null;
  const decoder = new TextDecoder();

  // 2. 逐 chunk 处理
  for await (const chunk of upstream.body as AsyncIterable<Buffer>) {
    // 2a. 立即转发（这是第一优先级）
    clientRes.write(chunk);
    if (!firstTokenTime) firstTokenTime = Date.now();

    // 2b. 顺便 peek（try-catch 保证不阻塞）
    const text = decoder.decode(chunk, { stream: true });
    for (const line of text.split('\n')) {
      if (!line.startsWith('data: ') || line.includes('[DONE]')) continue;
      try {
        const json = JSON.parse(line.slice(6));
        const partial = extractUsageByProvider(json, provider);
        if (partial) capturedUsage = capturedUsage ? { ...capturedUsage, ...partial } : partial;
      } catch { /* 跳过 */ }
    }
  }

  clientRes.end();

  // 3. 流结束，回调写库
  const ttft = firstTokenTime ? firstTokenTime - startTime : null;
  onRequestEnd(capturedUsage, ttft);
}

// 统一分发入口
function extractUsageByProvider(json: any, provider: string): CapturedUsage | null {
  switch (provider) {
    case 'anthropic': return extractAnthropicUsage(json);
    case 'gemini': return extractGeminiUsage(json);
    default: return extractOpenAIUsage(json);  // OpenAI + 兼容格式
  }
}

Layer 4：多厂商适配 — 从"能用"到"通用"
为什么第四做厂商适配
此时你已经有了 OpenAI 格式的提取逻辑。但各家厂商的 usage 字段名不同、出现位置不同。适配层就是把"不同"翻译成"统一"。
开发顺序：先 OpenAI 兼容（覆盖面最大），再 Anthropic（最复杂），最后 Gemini
第 1 步：写 providers/openai.ts — 覆盖 4 个厂商
import { CapturedUsage } from '../../types';

// OpenAI / DeepSeek / 智谱 / 通义 — 都用这个格式
export function extractOpenAIUsage(json: any): CapturedUsage | null {
  if (!json.usage) return null;
  const u = json.usage;
  
  return {
    prompt_tokens: u.prompt_tokens ?? 0,
    completion_tokens: u.completion_tokens ?? 0,
    total_tokens: u.total_tokens ?? 0,
    
    // OpenAI 缓存 token 在嵌套对象里
    cache_read_tokens: u.prompt_tokens_details?.cached_tokens ?? 0,
    cache_creation_tokens: 0,  // OpenAI 没有这个字段
    
    // DeepSeek 特殊字段：和 OpenAI 的字段名不同
    ...(u.prompt_cache_hit_tokens !== undefined && {
      cache_read_tokens: u.prompt_cache_hit_tokens,
    }),
    ...(u.prompt_cache_miss_tokens !== undefined && {
      cache_creation_tokens: u.prompt_cache_miss_tokens,
    }),
    
    cache_creation_5m_tokens: 0,
    cache_creation_1h_tokens: 0,
  };
}

关键思路： 一个文件覆盖 4 个厂商。因为 DeepSeek/智谱/通义用的就是 OpenAI 格式，区别只是多了几个额外字段。用 ?? 和展开运算符合并，不需要 if/else 判断具体是哪家。

import { CapturedUsage } from '../../types';

// Anthropic 的 usage 分散在两个事件里，必须合并
export function extractAnthropicUsage(json: any): CapturedUsage | null {
  // 事件 1: message_start — 包含 input_tokens + cache 字段
  if (json.type === 'message_start' && json.message?.usage) {
    const u = json.message.usage;
    return {
      prompt_tokens: u.input_tokens ?? 0,
      completion_tokens: 0,       // 此时还没有输出
      total_tokens: u.input_tokens ?? 0,
      cache_read_tokens: u.cache_read_input_tokens ?? 0,
      cache_creation_tokens: u.cache_creation_input_tokens ?? 0,
      cache_creation_5m_tokens: 0,
      cache_creation_1h_tokens: 0,
    };
  }
  
  // 事件 2: message_delta — 补充 output_tokens
  if (json.type === 'message_delta' && json.usage) {
    return {
      prompt_tokens: 0,
      completion_tokens: json.usage.output_tokens ?? 0,
      total_tokens: json.usage.output_tokens ?? 0,
      cache_read_tokens: 0,
      cache_creation_tokens: 0,
      cache_creation_5m_tokens: 0,
      cache_creation_1h_tokens: 0,
    };
  }
  
  return null;
}

关键思路： 尾捕获时，message_start 和 message_delta 的 usage 会被 { ...capturedUsage, ...partial } 合并。message_start 提供 prompt_tokens + cache，message_delta 补上 completion_tokens。最终结果是一个完整的 usage。

import { CapturedUsage } from '../../types';

export function extractGeminiUsage(json: any): CapturedUsage | null {
  const meta = json.usageMetadata;
  if (!meta) return null;
  
  // 🚨 关键：Gemini 的 promptTokenCount 包含 cachedContentTokenCount
  // 参考 Sub2API: stream_transformer.go L112-L118
  const cached = meta.cachedContentTokenCount ?? 0;
  
  return {
    prompt_tokens: meta.promptTokenCount - cached,  // 必须减掉
    completion_tokens: meta.candidatesTokenCount ?? 0,
    total_tokens: (meta.promptTokenCount ?? 0) + (meta.candidatesTokenCount ?? 0),
    cache_read_tokens: cached,
    cache_creation_tokens: 0,
    cache_creation_5m_tokens: 0,
    cache_creation_1h_tokens: 0,
  };
}

第 1 步：写聚合 SQL — src/db/aggregation.sql
-- 每小时执行一次，把 usage_logs 聚合到 token_hourly_stats

INSERT INTO token_hourly_stats (hour, provider, model, 
  prompt_tokens, completion_tokens, total_tokens,
  cache_read_tokens, cache_creation_tokens,
  input_cost, output_cost, total_cost,
  request_count, avg_latency_ms, avg_first_token_ms)
SELECT 
  date_trunc('hour', created_at) AS hour,
  provider, model,
  SUM(prompt_tokens), SUM(completion_tokens), SUM(total_tokens),
  SUM(cache_read_tokens), SUM(cache_creation_tokens),
  SUM(input_cost), SUM(output_cost), SUM(total_cost),
  COUNT(*), AVG(latency_ms)::int, AVG(first_token_ms)::int
FROM usage_logs
WHERE created_at >= date_trunc('hour', NOW() - INTERVAL '2 hours')
  AND created_at < date_trunc('hour', NOW())
GROUP BY hour, provider, model
ON CONFLICT (hour, provider, model) DO UPDATE SET
  prompt_tokens = EXCLUDED.prompt_tokens,
  completion_tokens = EXCLUDED.completion_tokens,
  total_tokens = EXCLUDED.total_tokens,
  cache_read_tokens = EXCLUDED.cache_read_tokens,
  cache_creation_tokens = EXCLUDED.cache_creation_tokens,
  input_cost = EXCLUDED.input_cost,
  output_cost = EXCLUDED.output_cost,
  total_cost = EXCLUDED.total_cost,
  request_count = EXCLUDED.request_count,
  avg_latency_ms = EXCLUDED.avg_latency_ms,
  avg_first_token_ms = EXCLUDED.avg_first_token_ms,
  updated_at = NOW();

第 2 步：写定时调度 — src/core/scheduler.ts
import cron from 'node-cron';
import pool from '../db/connection';
import fs from 'fs';

const aggregationSQL = fs.readFileSync('./src/db/aggregation.sql', 'utf8');

export function startScheduler() {
  // 每小时第 5 分钟执行（给 usage 写入留缓冲）
  cron.schedule('5 * * * *', async () => {
    try {
      await pool.query(aggregationSQL);
      console.log('[scheduler] hourly aggregation completed');
    } catch (err) {
      console.error('[scheduler] aggregation failed:', err);
    }
  });
}

第 4 步：写核心查询路由 — src/api/routes/usage.ts
import { Router } from 'express';
import pool from '../../db/connection';

const router = Router();

// 仪表板首页大数字
router.get('/summary', async (req, res) => {
  const period = req.query.period || '30d';
  const result = await pool.query(`
    SELECT 
      SUM(total_cost) as total_cost,
      SUM(prompt_tokens + completion_tokens) as total_tokens,
      COUNT(DISTINCT model) as active_models,
      COALESCE(SUM(cache_read_tokens)::float / NULLIF(SUM(cache_read_tokens + prompt_tokens), 0), 0) as cache_hit_rate
    FROM usage_logs 
    WHERE created_at >= NOW() - INTERVAL $1
  `, [period]);
  
  res.json(result.rows[0]);
});

// Token 趋势
router.get('/trend', async (req, res) => {
  const period = req.query.period || '7d';
  const granularity = req.query.granularity || 'hour';
  
  const result = await pool.query(`
    SELECT 
      date_trunc($2, hour) as time,
      SUM(prompt_tokens) as prompt_tokens,
      SUM(completion_tokens) as completion_tokens,
      SUM(total_cost) as total_cost
    FROM token_hourly_stats
    WHERE hour >= NOW() - INTERVAL $1
    GROUP BY time ORDER BY time
  `, [period, granularity]);
  
  res.json(result.rows);
});

// 按模型成本
router.get('/by-model', async (req, res) => {
  const period = req.query.period || '30d';
  const result = await pool.query(`
    SELECT model, provider,
      SUM(prompt_tokens) as prompt_tokens,
      SUM(completion_tokens) as completion_tokens,
      SUM(input_cost) as input_cost,
      SUM(output_cost) as output_cost,
      SUM(total_cost) as total_cost
    FROM usage_logs
    WHERE created_at >= NOW() - INTERVAL $1
    GROUP BY model, provider
    ORDER BY total_cost DESC
  `, [period]);
  
  res.json(result.rows);
});

// 缓存命中率
router.get('/cache-stats', async (req, res) => {
  const period = req.query.period || '7d';
  const result = await pool.query(`
    SELECT model, provider,
      SUM(cache_read_tokens) as cache_read_tokens,
      SUM(cache_creation_tokens) as cache_creation_tokens,
      SUM(prompt_tokens) as prompt_tokens,
      CASE WHEN SUM(prompt_tokens + cache_read_tokens) > 0
        THEN SUM(cache_read_tokens)::float / SUM(prompt_tokens + cache_read_tokens)
        ELSE 0 END as cache_hit_rate
    FROM usage_logs
    WHERE created_at >= NOW() - INTERVAL $1
    GROUP BY model, provider
    ORDER BY cache_hit_rate DESC
  `, [period]);
  
  res.json(result.rows);
});

export default router;

第 4 步：写首页 — src/pages/Dashboard.tsx
开发顺序：先大数字 → 再趋势图 → 再饼图
import { Card, Metric, Text, BarChart, DonutChart } from '@tremor/react';
import { fetchSummary, fetchTrend, fetchByProvider } from '../api/client';
import { useEffect, useState } from 'react';

export default function Dashboard() {
  const [summary, setSummary] = useState<any>(null);
  const [trend, setTrend] = useState<any[]>([]);
  const [providers, setProviders] = useState<any[]>([]);

  useEffect(() => {
    fetchSummary('30d').then(r => setSummary(r.data));
    fetchTrend('7d', 'hour').then(r => setTrend(r.data));
    fetchByProvider('30d').then(r => setProviders(r.data));
  }, []);

  if (!summary) return <div>Loading...</div>;

  return (
    <div className="p-6 space-y-6">
      {/* 大数字卡片 */}
      <div className="grid grid-cols-4 gap-4">
        <Card>
          <Text>本月总消耗</Text>
          <Metric>${Number(summary.total_cost).toFixed(2)}</Metric>
        </Card>
        <Card>
          <Text>总 Token</Text>
          <Metric>{Number(summary.total_tokens).toLocaleString()}</Metric>
        </Card>
        <Card>
          <Text>活跃模型</Text>
          <Metric>{summary.active_models}</Metric>
        </Card>
        <Card>
          <Text>缓存命中率</Text>
          <Metric>{(Number(summary.cache_hit_rate) * 100).toFixed(1)}%</Metric>
        </Card>
      </div>

      {/* Token 趋势图 */}
      <Card>
        <Text>Token 消耗趋势 (7天)</Text>
        <BarChart data={trend} index="time" categories={['prompt_tokens', 'completion_tokens']} />
      </Card>

      {/* 厂家饼图 */}
      <Card>
        <Text>厂家消耗分布</Text>
        <DonutChart data={providers} index="provider" category="total_cost" />
      </Card>
    </div>
  );
}
